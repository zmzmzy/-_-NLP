# 系统架构说明（v0.1）

## 分层设计

1. 前端层（Vue）
2. 后端 API 层（C++ Drogon）
3. 数据层（MySQL）
4. 匹配引擎层（NLP 与规则融合）

## 主要数据流

1. 管理员维护学院、专业、企业、岗位、学生与就业记录。
2. 匹配引擎读取结构化字段与文本字段。
3. 引擎计算语义分、技能分、约束分并融合成最终得分。
4. 匹配解释结果写入 `match_results`。
5. 决策看板按专业聚合 `employment_records + match_results`。

## 功能模块

- 用户与认证
- 系统日志
- 主数据管理
- 人岗匹配服务
- 专业决策统计看板

## 非功能目标

- 常规 CRUD 接口响应时间 < 300ms
- `match/single` 在 v0.1 基线模式下响应 < 500ms
- 管理端关键操作可追溯
- 实验流程可复现
