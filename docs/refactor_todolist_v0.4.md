# 重构 TODO（v0.4：学生就业-专业对口主线）

目标：将系统主流程收敛为“专业 -> 学生 -> 就业 -> 匹配 -> 专业决策”，满足导师提出的 3 条核心要求，并形成可二次迭代的稳定工程基线。

执行规则（必须遵守）：

1. 严格按阶段顺序推进，不跳步。
2. 每个任务必须有“验收标准（DoD）”才算完成。
3. 每轮迭代最多推进 2-4 个任务，完成后立即回归测试。
4. 不做大爆炸重构，优先增量改造与兼容已有数据。
