# 需求基线（冻结版）

日期：2026-03-02

本文档将指导老师要求转化为可实现、可验收的系统需求。

## R1 就业信息维度

系统中的岗位信息至少包含：

- 企业名称
- 企业主业
- 企业拳头产品
- 产品所属领域
- 产品应用行业
- 岗位名称
- 岗位职责文字描述
- 岗位技术项目文字描述
- 岗位专业知识要求
- 岗位专业技能要求
- 岗位技术平台/工具要求
- 硬约束（学历、地点等）

## R2 专业信息维度

系统中的专业信息至少包含：

- 学院名称
- 专业名称
- 专业核心课程文字描述
- 专业知识与技能文字描述
- 专业技术平台需求文字描述
- 专业能力培养文字描述

数据集最低要求：

- 覆盖 10-20 个不同学院
- 各学院下有多个专业，不能仅单学院构造样本

## R3 匹配与决策维度

系统必须支持：

- 就业岗位信息与专业信息的文本关联匹配
- 学生级匹配得分与可解释结果
- 专业级聚合统计：
  - 专业平均匹配分
  - 专业就业对口率
  - 专业技能缺口分布
- 为专业建设与招生方向提供数据支持

## R4 工程实现要求

- 提供核心实体 RESTful CRUD 接口
- 使用 MySQL 并保证关系完整性
- 保留用户系统与系统日志
- 采用前后端分离架构
- 支持可复现测试和实验评估

## R5 实验评估要求

论文与系统至少产出：

- 基线模型与改进模型对比
- 指标：Precision@5、Precision@10、NDCG@10
- 消融实验：
  - 去掉技能加权
  - 去掉硬约束
